虚拟服务配置手册

v.1.0.0

作用:

  1. 虚拟服务是一个可以接收任意请求路径请求的应答器,并可按照配置返回结果或发起http调用.

 

配置:

  1. 配置示例

    1. 调用配置

      1. 调用时需要将域名切换为:https://speedhelper-api-dev.gtech.asia
      2. 控制台地址:https://speedhelper-api-dev.gtech.asia

    2. 虚拟服务配置

      {
          "keyUrl": "/silwings",
          "param": {
                  "name": "Silwings",
                  "age": "100",
                  "orderId":"OD0001",
                  "descendant":{
                      "son":"小小明"
                  }
          },
          "logic": {
              "logic": "name == Silwings & age == 10 return result01\nname == Silwings & age != 100 throw 这是异常信息!!!\nname == Silwings & age == 100 return result03"
          },
          "callback": {
              "body": {
                  "paymentCallback": {
                      "param01": "value01",
                      "param02": {
                          "orderId": "${orderId}"
                      }
                  }
              },
              "callbackUrl": "http://127.0.0.1:8899/payment/notify",
              "method": "POST",
              "param": ""
          },
          "result": {
              "result01": "这是返回值01",
              "result02": "这是返回值02",
              "result03": "这是返回值03",
              "default": {
                  "key": "这是默认返回值."
              }
          },
          "type": "BASE"
      }

     

  2. 示例说明

    1. keyUrl:

      1. 需要模拟的url地址.需要带"/",必填

    2. param:

      1. 该地址的入参格式.用于逻辑对照.可以不填写

    3. type 可选值,默认为 BASE:

      1. 可选值(大小写敏感):

        1. BASE: 完全匹配keyUrl配置的地址
        2. REST_FUL: 通过startsWith匹配url地址.如keyUrl配置为"/abc/",向"/abc/efg"发起请求将命中

      2. 说明:

        1. 同一个url在BASE和REST_FUL中各可以存在一个,同时存在时优先匹配BASE

    4. result 可以配置多个预设的返回值:

      1. 接收键值对数据
      2. key固定为字符串,value可以为字符或对象
      3. 最终会通过logic的计算结果决定哪一个result将会被返回
      4. 使用关键字default可以配置默认返回值.如果logic没有匹配到任何结果会使用default返回
      5. 在没有设置default,logic又没有匹配到任何值时,将返回一个中文描述信息
      6. 如果logic条件判断成功但没有找到对应的result选项,会返回设置的result名称值
      7. 如果没有配置logic,也没有配置default,将返回null
      8. 支持UUID()和CB{}的内置函数

    5. logic 可以配置多条简单的条件语句:

      1. 示例: 

        xxxxxxxxxx
        // 如果请求参数中name字段的值 equals Silwings 并且 age 段的值 equals 10,
        // 返回result01 的结果
        name == Silwings & age == 10 return result01

      2. 说明:

        1. logic中比较的实质是请求参数是否满足某些条件
        2. 比较符号左边对应请求参数中的参数名,比较符号右边对应请求参数的值.比较符号:==,!=
        3. logic通过"行"来分割,计算时会按照行顺序依次遍历,当有第一个满足条件的logic出现时立即返回

      3. 关键字

        1. 关键字列表

          1. == : 等于equals(),无论是数值还是字符均以字符串进行比较
          2. != : 等于equals()取反,无论是数值还是字符均以字符串进行比较
          3. & : 等于&,并且
          4. return : 标记要返回的结果
          5. throw : 标记抛出的异常信息
          6. \n: 换行符

        2. 注意事项

          1. throw: 和return一样都是起返回的作用.但return的返回值会在result中查找,而throw是直接返回throw后面的字符.另外在存在回调的场景下,如果logic中throw语句条件命中,将不进行回调处理
          2. 除了"\n",其他关键字两边必须含有空格

    6. callback: 回调

      1. 描述:

        1. 通过配置该属性,虚拟服务在接收并成功处理请求后,经过短暂延迟会向指定地址发起请求

      2. 字段说明:

        1. callbackUrl: 回调地址,需要配置完整地址,如: http://127.0.0.1:8899/payment/notify

        2. method:

          1. 描述:

            1. 发起回调的请求方式,目前只支持POST请求,请输入POST.

          2. 可选值(大小写敏感):

            1. POST

        3. param: 拼接在url中的参数,目前暂不支持,请直接填写空串.

        4. body:

          1. 描述:

            1. 发起回调时的POST请求请求体
            2. 可配置无限层级的对象类型
            3. 支持${}和UUID()内置函数

  3. 内置函数

    1. 内置函数是通过指定格式的字符标识可以进行字符替换的工具方法

    2. 只有支持内置函数的领域写内置函数才能生效.如callback

    3. 目前支持的内置函数(大小写敏感):

      1. ${}
      2. CB{}
      3. UUID()

    4. 函数说明:

      1. ${}:

        1. 从请求参数中将指定key的值在标记处替换.如${name},如果请求参数中包含name等于"小明",${name}将被替换为小明.如果不存在将保持原样.
        2. 只支持一级请求参数.对于{"name","小明","age":"10","descendant":{"son":"小小明"}这样的参数,使用${descendant},可以替换为{"son":"小小明"},但如果使用${son},将保持原样.

      2. CB{}:

        1. 用于从callback获取属性值.用法和${}相同.示例:callback中有param:UUID(),返回值中包含CB{param},将会使用param中用uuid生成的实际值替换CB{param}
        2. 仅支持一级参数.

      3. UUID():

        1. 该标识无论出现在哪个参数层级,都会被替换为一个去除了"-"的UUID随机数值.

  4. 模拟调用

    1. 根据配置示例中的配置,使用不同的参数发起请求
